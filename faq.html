자주 묻는 질문

FREQUENTLY ASKED QUESTIONS

이 페이지는 "로또 6/45 번호 분석기" 와 관련해 이용자로부터 받은, 혹은 받을 것으로 예상되는 질문을 범주별로 정리한 FAQ 입니다. 각 답변은 통계학·확률론의 기본 원리에 근거해 작성되었으며, 사실 관계는 공개된 자료를 통해 교차 검증이 가능합니다.

① 알고리즘과 방법론

1. 이 도구가 추천한 번호로 당첨 확률이 올라가나요?

올라가지 않습니다. 로또 6/45 의 1 등 당첨 확률은 1 / 8,145,060 이며, 이 수치는 어떤 번호 선택 방식(자동·수동·분석·직관) 을 사용해도 변하지 않습니다. 본 도구는 "예측 도구" 가 아니라 "과거 빈도를 시각화하는 교육용 도구" 이며, 그 사실을 모든 페이지에서 투명하게 공개하고 있습니다.

2. 그럼 왜 "자주 나오는 번호" 를 가중치로 쓰나요?

가중치 추첨은 "지금까지 많이 나온 번호로 구성된 조합이 어떤 모습일까" 를 시각적으로 보여주기 위한 편향된 샘플링 기법입니다. 이것은 미래 결과에 대한 예측이 아니라, 과거 데이터의 모양을 탐색하는 한 가지 방법일 뿐입니다. 통계학에서 이런 방식은 "계층화 추출" 이나 "중요도 샘플링" 과 같은 정식 기법의 단순화 버전에 해당합니다.

3. 왜 한 세트에 6 개씩 5 세트를 주나요?

로또 6/45 는 1 게임당 6 개 번호를 고르는 구조이므로 세트당 6 개는 고정입니다. 5 세트인 이유는 한 장의 복권 용지(실제 판매되는 인쇄물) 가 A ~ E 의 5 개 게임을 담는 형식이기 때문입니다. 이 도구는 "한 장 분량의 조합" 을 한 번에 보여주는 UI 컨벤션을 따랐습니다.

4. "자주 나오는 번호 추첨" 과 "가중치 추첨" 의 차이는?

가중치 추첨은 가중치 풀에서 한 번 뽑아 5 세트를 만들어 줍니다. "자주 나오는 번호 추첨" 은 같은 가중치 풀로 뽑되, 점수(= 조합 평균 빈도) 가 70 점 이상인 조합만 걸러내 5 세트로 구성합니다. 즉 후자는 "빈도가 높은 번호 위주" 로만 모인 조합을 보장합니다. 결과의 확률적 의미는 동일합니다.

5. 가중치 공식이 정확히 어떻게 되나요?

각 번호의 출현 빈도 f(n) 을 (f(n) - minF) / (maxF - minF) 로 0 ~ 1 사이로 정규화한 뒤, round(p × 4) + 1 로 1 ~ 5 사이의 정수 가중치로 변환합니다. 이 가중치만큼 번호를 풀(pool) 에 반복해 넣고 무작위로 뽑습니다. 자세한 단계는 사이트 소개 를 참고해 주세요.

② 데이터와 신뢰성

6. 데이터는 어디서 가져왔나요?

역대 약 1,161 회차까지의 당첨 번호 빈도를 자바스크립트 객체로 스크립트에 내장했습니다. 이 데이터는 최신 회차를 자동으로 반영하지 않으며, 특정 시점에 고정되어 있습니다. 최신 당첨 번호는 동행복권 공식 홈페이지 에서 확인하실 수 있습니다.

7. 왜 실시간 API 를 연동하지 않았나요?

본 도구는 정적 GitHub Pages 에서 운영되는 무료 서비스이며, 백엔드 서버나 유료 API 키 관리가 필요 없도록 설계되었습니다. 정적 데이터만으로도 "가중치 알고리즘이 실제로 어떻게 동작하는지" 를 학습하고 체험하기에는 충분합니다.

8. 데이터가 정확한지 어떻게 확인하나요?

모든 원본 데이터는 GitHub 저장소의 main.js 파일 상단 freq 객체에 공개되어 있습니다. 누구나 직접 열람·복사해 독립적인 집계 도구(Python, R, Excel 등) 로 재검증할 수 있습니다. 이것이 오픈소스로 제공되는 이유입니다.

③ 확률론 개념

9. 도박사의 오류가 뭔가요?

독립 사건에서 "과거의 결과가 미래의 확률에 영향을 준다" 고 잘못 믿는 인지 편향입니다. 예: "지난 10 회차 동안 7 번이 한 번도 안 나왔으니 이번엔 나올 확률이 높다." 틀렸습니다. 각 회차는 독립적이며 7 번이 다음 회차에 뽑힐 확률은 이전 결과와 무관하게 6/45 ≈ 13.3% 입니다.

10. "조합 확률이 같다" 는 게 정말인가요?

네, 수학적으로 완전히 같습니다. 1-2-3-4-5-6 이든 11-17-23-29-34-41 이든, 특정한 6 개 조합이 뽑힐 확률은 정확히 1 / 8,145,060 입니다. 차이가 나는 것은 "뽑힐 확률" 이 아니라 "당첨됐을 때 그 조합을 함께 고른 다른 사람의 수" 입니다. 많은 사람이 동시에 고르는 "흔한" 조합은 당첨금이 더 많이 쪼개집니다.

11. 대수의 법칙이란 뭔가요?

"시행 횟수가 많아질수록 관찰 비율이 이론 비율에 가까워진다" 는 통계학의 기본 정리입니다. 로또에서 이 법칙은 "회차가 많이 쌓일수록 각 번호의 출현 빈도가 서로 비슷해진다" 는 형태로 나타납니다. 단기적으로 어떤 번호가 많이/적게 나오는 것은 정상적인 무작위 변동입니다.

12. 기대값(expected value) 으로 보면 로또는 어떤가요?

1,000 원짜리 티켓 한 장의 장기 평균 기대값은 대략 400 ~ 600 원 구간에 위치합니다. 즉 "평균적으로 1,000 원 내고 절반 정도를 돌려받는 게임" 입니다. 이 손실분(40 ~ 60%) 은 운영비, 조세, 공익 환원금으로 사용됩니다. 기대값 관점에서 로또는 "투자 수단" 이 아니라 "오락 지출 + 공익 기부" 입니다.

④ 개인정보와 광고

13. 개인정보를 수집하나요?

아니요. 본 사이트는 회원가입, 로그인, 이메일 수집 기능 자체가 없으며, 어떤 개인정보도 서버로 전송하지 않습니다. 모든 번호 생성은 이용자의 브라우저 안에서만 처리됩니다. 자세한 내용은 개인정보처리방침 을 참고해 주세요.

14. 광고가 표시되나요?

운영 비용 충당을 위해 Google AdSense 광고를 게재합니다. Google 및 제3자 공급업체는 이전 방문 정보 기반의 맞춤 광고를 위해 쿠키(특히 DART 쿠키) 를 사용할 수 있으며, 이용자는 Google 광고 설정 에서 맞춤 광고를 직접 해제할 수 있습니다.

15. 쿠키를 거부하고 싶어요.

브라우저 설정(Chrome: 설정 → 개인정보 및 보안 → 쿠키 / Firefox: 설정 → 개인정보 및 보안 → 쿠키 및 사이트 데이터) 에서 서드파티 쿠키를 차단할 수 있습니다. 다만 이 경우 일부 광고가 맞춤형에서 일반형으로 바뀌거나 웹사이트 기능의 일부가 제한될 수 있습니다.

⑤ 기술과 소스코드

16. 어떤 기술로 만들었나요?

프론트엔드 스택은 Vanilla HTML + CSS + JavaScript 이며, 프레임워크나 빌드 단계가 없습니다. 배포는 GitHub Pages, CI 는 GitHub Actions 를 사용합니다. 외부 의존성은 Google Fonts(Noto Sans KR, Black Han Sans) 와 Google AdSense 로더 뿐입니다.

17. 소스 코드를 볼 수 있나요?

네. 전체 소스 코드는 GitHub 저장소 에서 공개되어 있습니다. MIT 비슷한 자유 이용 전제로, 누구나 열람·학습·기여할 수 있습니다.

18. 이 도구를 내 블로그에 임베드할 수 있나요?

소스 코드가 공개되어 있으므로 본인의 사이트에 동일한 기능을 직접 구축하는 것은 가능합니다. 다만 이 사이트 자체를 iframe 으로 불러오는 행위는 권장하지 않습니다 — AdSense 정책과 GitHub Pages 이용약관을 우회하는 것으로 간주될 수 있습니다.

⑥ 책임과 법적 고지

19. 이 도구로 당첨되면 누가 세금을 내나요?

본인입니다. 본 도구는 단지 번호 조합을 보여줄 뿐이며, 실제 복권 구매와 당첨금 수령은 전적으로 이용자 본인이 수행합니다. 3 억 원 초과 당첨금은 약 33%(지방소득세 포함 33%) 가 원천징수됩니다.

20. 미성년자도 사용할 수 있나요?

이 도구의 사용 자체에는 연령 제한이 없지만, 복권 구매는 만 19 세 이상 만 가능합니다. 미성년자가 이 사이트에서 생성된 번호로 복권을 구매하는 것은 법적으로 금지되어 있으며, 본 사이트는 그러한 사용을 권장하지 않습니다.

21. 도박 문제로 어려움을 겪고 있어요.

한국도박문제예방치유원(☎ 1336) 에서 24 시간 상담과 치유 프로그램을 무료로 제공합니다. 온라인으로는 kcgp.or.kr 에서 자가진단과 전문 상담 신청이 가능합니다. 도박 문제는 조기에 상담을 받을수록 회복 가능성이 높으니 망설이지 말고 연락해 주세요.

22. 질문을 추가하거나 수정 제안을 하려면?

사이트 전체가 오픈소스이므로 GitHub 저장소 이슈 페이지 에서 질문, 수정 제안, 버그 리포트를 남겨 주실 수 있습니다. 답변 드릴 수 있는 질문은 이 FAQ 페이지에 반영됩니다.
