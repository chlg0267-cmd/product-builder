ABOUT THIS TOOL

무엇을 하는 도구인가요

로또 6/45 번호 분석기 는 국내 복권 "로또 6/45" 의 역대 당첨 번호를 통계적으로 분석해, 각 번호가 지금까지 얼마나 자주 나왔는지 시각화하고 그 빈도에 기반한 가중치 추첨 알고리즘으로 번호 조합을 제안하는 웹 기반 무료 도구입니다. 설치가 필요 없고, 어떤 개인정보도 수집하지 않으며, 사용자의 브라우저 안에서만 동작합니다.

이 도구의 목적은 "당첨을 예측하는 것"이 아닙니다. 누구나 알 수 있는 엄밀한 수학적 사실은 — 로또는 완전한 무작위 추첨이며 어떤 알고리즘도 당첨 번호를 예측할 수 없다 — 는 것입니다. 그럼에도 불구하고 "그동안 어떤 번호가 많이 나왔는가", "빈도 분포는 어떤 모양인가" 라는 질문은 통계적으로 흥미로운 주제이며, 이 도구는 그 통계를 누구나 직관적으로 확인할 수 있게 해주는 교육·오락용 시각화 도구입니다.

사용한 데이터

현재 버전은 로또 6/45 역대 약 1,161회차 당첨 번호 의 빈도 집계를 내장된 정적 JavaScript 객체(freq) 로 사용합니다. 이 수치는 각 번호가 당첨 번호로 선택된 누적 횟수를 의미하며, 1 번부터 45 번까지 약 158 회에서 197 회 사이의 범위를 가집니다. 평균적으로 모든 번호가 비슷한 빈도로 나오지만, 약 40 회 내외의 편차가 관찰됩니다.

본 도구는 실시간 당첨 번호 API 를 연동하고 있지 않으므로, 데이터는 특정 시점에 고정되어 있으며 이후 추첨 결과는 반영되지 않습니다. 최신 당첨 번호를 확인하고 싶으시면 동행복권 공식 홈페이지 를 방문해 주세요.

알고리즘 — 가중치 추첨

"로또번호 추첨 받기" 버튼은 다음과 같이 동작합니다.

  1. 정규화: 1 번부터 45 번까지의 출현 횟수 freq[n] 을 최소값 minF 와 최대값 maxF 사이에서 0.0 ~ 1.0 범위로 정규화합니다.
  2. 가중치 계산: 각 번호의 정규화된 값을 0 ~ 4 구간의 정수 가중치 w 로 변환합니다. 공식은 다음과 같습니다.
w(n) = round((freq[n] − minF) / (maxF − minF) × 4) + 1

즉, 자주 나온 번호일수록 w 가 최대 5 까지 커지고, 드문 번호는 1 에 가까워집니다.

  1. 풀 구성: 번호 n 을 가중치 w(n) 만큼 반복해 풀(pool) 에 넣습니다. 예를 들어 w(18) = 5 라면 18 은 풀에 5 번 들어갑니다.
  2. 무작위 추출: 풀에서 무작위로 한 개를 뽑습니다. 이 과정을 중복 없이 6 번 반복해 한 세트를 만들고, 다시 5 세트가 될 때까지 반복합니다.
  3. 점수 계산: 생성된 6 개 번호의 평균 빈도를 minF / maxF 로 재정규화해 0 ~ 100 점수로 표시합니다. 점수가 높을수록 "역대 데이터에서 자주 나온 번호들의 조합"에 가깝다는 의미입니다.

"자주 나오는 번호 추첨" 버튼은 동일한 가중치 추첨을 반복하되, 점수 70 점 이상인 조합만 필터링 해 결과를 반환합니다. 즉 역대 빈도가 높은 번호 위주로 구성된 조합만 뽑히게 됩니다.

알고리즘의 한계점 (중요)

이 부분을 반드시 읽어주세요.

위 알고리즘은 과거 통계에 "편향된" 번호 조합을 만드는 방법 이지, 미래 당첨 번호를 예측하는 방법이 아닙니다. 로또 추첨기는 매 회차 완벽하게 독립적인 무작위 시행이며, 이전 회차의 당첨 번호가 다음 회차의 결과에 영향을 주지 않습니다. 이것을 통계학에서는 "사건의 독립성" 이라고 부릅니다.

따라서:

  • 자주 나온 번호가 앞으로도 자주 나올 확률은 덜 나온 번호가 앞으로 나올 확률과 수학적으로 동일 합니다.
  • 어떤 조합(예: 1-2-3-4-5-6) 이든 다른 조합과 당첨 확률은 정확히 같습니다 — 약 1/8,145,060.
  • 본 도구가 생성한 번호로 당첨되었든 직접 고른 번호로 당첨되었든, 그 결과는 순전히 우연입니다.

그럼에도 이 도구가 제공하는 가치는 "빈도 데이터를 시각적으로 탐색하고, 빈도에 기반한 조합 생성 알고리즘을 이해해 보는 학습 경험" 입니다.

개인정보 처리

이 도구는 사용자의 어떤 개인정보도 수집·저장·전송하지 않습니다. 모든 번호 생성 로직은 브라우저 안에서 실행되며, 서버와의 통신은 정적 자산(HTML, CSS, JS, 폰트) 다운로드 외에는 존재하지 않습니다. 자세한 내용은 개인정보처리방침 을 참고해 주세요.
